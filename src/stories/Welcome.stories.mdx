# 信使前端框架

信使是基于 Material UI 的 Angular 前端框架，后端适配 Drupal，提供优秀的数字创新体验。

- 为了适配 IE11 目前的版本保持在 Angular v11；
- 支持 SSR 服务端渲染；
- 支持多主题、暗黑模式；
- 全局配置、品牌菜单配置、主题配置、API、用户状态在 app.module 中注入，不同的项目可覆盖 Token 改变业务逻辑；
- 页面内容数据通过当前页面的 url 来读取 api，根据返回的 body 数组渲染对应的组件；
- 框架全面使用 Storybook 进行开发、测试和文档的更新；

## 主要功能

- 登录页支持手机号和用户名密码登录方式；
- 地图使用高德地图，切换主题时支持匹配暗黑和浅色风格；
- 可以设置强制弹窗提示信息；
- 支持谷歌 Analytics 分析；
- 支持开启或者禁用全站权限访问；
- 支持站点消息推送；
- 链接或者组件可根据角色显示；
- 页头和页脚根据 JSON 构建，菜单支持角色显示；
- 动态组件，动态表单

## 技术选型

- 前端：Angular + Material + FlexLayout
- 动态表单：ngx-formly
- 图表：Echarts
- pdf 生成：jspdf
- 编辑器：quill
- 幻灯片：swiper
- 加密：crypto-js
- 工具函数：lodash-es
- 测试预览：storybook
- web 服务：Nginx
- 后端：Drupal(推荐)

## 缓存方案

用户访问数据会经过多层缓存：

1. ngxin proxy 反向代理缓存，有缓存直接返回，通过 url 参数或者清空缓存文件夹可查看最新；
2. app 程序内请求缓存，刷新页面或者重新加载可查看最新；
3. api 后端缓存，清空后台缓存；
